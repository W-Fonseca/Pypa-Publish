Connection Database

In order for the application to work with job queues, store codes and folders, and manage when a process will be executed, we need a connection to a database. So check which type of connection fits with your database.

Note: Remember that for communication with other machines and Task Scheduler management, your database must be visible.

SQL Server (SQL Authentication)

  • Data Source: Defines the name or IP address of the SQL Server. It may include a named instance.
  • Initial Catalog: Specifies the name of the database you want to connect to.
  • User Id: The username used for SQL Server authentication.
  • Password: The password associated with the User Id.
  • AdditionalSQL (Connection Parameters): Additional string to allow customization of the connection with specific parameters.

Usage Example: Connect to SQL Server with specific credentials, for example, in scenarios where access to SQL Server is restricted and Windows authentication cannot be used.

SQL Server (Windows Authentication)

  • Data Source: Name or IP address of the SQL Server.
  • Initial Catalog: Name of the database you want to connect to.
  • Integrated Security: Indicates that the connection should be made using the Windows credentials of the user running the application.
  • AdditionalSQL (Connection Parameters): Additional string for connection customization.

Usage Example: Connect to SQL Server in an environment where the user is already authenticated on the Windows domain, simplifying the authentication process and enhancing security.

Availability Group (SQL Authentication)

  • Server: Name of the SQL Server or listener associated with the availability group.
  • ListenerPort: Port used by the listener to connect to SQL Server.
  • Database: Name of the database you want to connect to.
  • MultiSubnetFailover: Indicates if failover should be handled in an environment with multiple subnets. This parameter improves reconnection in high-availability environments.
  • User Id: Username for SQL authentication.
  • Password: Password corresponding to the User Id.
  • AdditionalSQL (Connection Parameters): Additional string for connection customization.

Usage Example: Connect to an SQL Server configured with Availability Groups using SQL authentication, in critical environments where automatic failover is essential.

Availability Group (Windows Authentication)

  • Server: Name of the SQL Server or listener associated with the availability group.
  • ListenerPort: Port used by the listener to connect to SQL Server.
  • Database: Name of the database you want to connect to.
  • MultiSubnetFailover: Indicates if failover should be handled in an environment with multiple subnets.
  • Integrated Security: Connection using Windows credentials.
  • AdditionalSQL (Connection Parameters): Additional string for connection customization.

Usage Example: Connect to an SQL Server with Availability Groups using Windows authentication, ideal in corporate environments where security is managed by the Windows domain.

SQL Server (Custom Connection String)

  • Connection String: Custom connection string with detailed parameters for specific needs.
  • Usage: Adapt connection strings to your needs, considering options such as encryption, failover, and timeouts.

Usage Example: Use a customized connection string for special cases or advanced configurations not covered by standard methods.
